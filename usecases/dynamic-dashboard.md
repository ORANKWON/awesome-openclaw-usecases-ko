# 동적 대시보드

## 문제점
맞춤형 대시보드 구축에 몇 주가 걸리고, 여러 API를 순차적으로 폴링하면 느리고 속도 제한에 걸립니다.

## 기능 설명
병렬 서브에이전트를 활용한 실시간 대시보드:
- 각 서브에이전트가 하나의 데이터 소스 수집 (GitHub, Twitter, Polymarket, 시스템 상태)
- 결과를 통합 대시보드로 집계 (텍스트, HTML, 또는 Canvas)
- N분마다 업데이트, 임계값 초과 시 알림
- 데이터베이스에 과거 지표 저장

**대시보드 섹션 예시:** GitHub 스타/포크/이슈, 소셜 미디어 언급/감성 분석, Polymarket 거래량, 시스템 CPU/메모리/디스크

## 설정 프롬프트

```
당신은 내 동적 대시보드 관리자입니다. 15분마다 크론 작업을 실행하여:

1. 다음 소스에서 데이터를 병렬로 가져오는 서브에이전트 실행:
   - GitHub: 스타, 포크, 오픈 이슈, 커밋 (지난 24시간)
   - Twitter: "@username" 언급, 감성 분석
   - Polymarket: 추적 중인 마켓의 거래량
   - 시스템: 셸 명령으로 CPU, 메모리, 디스크 사용량

2. 각 서브에이전트가 결과를 메트릭 데이터베이스에 기록.

3. 집계하여 대시보드 포맷 후 Discord #dashboard에 게시.

4. 알림 조건 확인 (스타 +50/시간, CPU >90%, 부정적 감성 급증).
```

## 필요한 스킬
- 서브에이전트 실행
- `github` CLI
- `bird` (Twitter)
- `web_search`
- `postgres`
- Discord 또는 Canvas
- 크론 작업
