# DenchClaw를 사용한 로컬 CRM 프레임워크

OpenClaw와 실제로 작동하는 CRM을 설정하는 것은 고통스럽습니다. 데이터베이스를 연결하고, UI를 구축하고, 브라우저 자동화를 설정하고, 메시징 플랫폼을 연결하고, 에이전트가 데이터 스키마를 이해하도록 해야 합니다. 대부분의 사람들은 중간에 포기하고 반쯤 작동하는 Notion 통합으로 끝납니다.

DenchClaw는 OpenClaw를 완전한 로컬 CRM, 세일즈 자동화 및 생산성 플랫폼으로 만드는 오픈소스 프레임워크입니다 — 단일 명령으로 설치하고 완전히 로컬 머신에서 실행됩니다.

## 문제점

OpenClaw는 프리미티브로서 매우 강력하지만, 실제 비즈니스 워크플로우(리드 추적, 아웃바운드, 파이프라인 관리)에 사용하려면 수십 가지 도구를 연결해야 합니다: 데이터베이스, UI, 브라우저 접근, 메시징 통합, 파일 관리. 새로운 통합마다 더 많은 수동 설정, 더 많은 자격증명 관리, 더 많은 고장이 발생합니다. Cursor 수준의 UX를 비즈니스 운영에 원하지, 쉘 스크립트 더미를 원하지 않습니다.

## 기능 설명

- **원커맨드 설정**: `npx denchclaw`가 모든 것을 설치합니다 — DuckDB 데이터베이스, 웹 UI, OpenClaw 프로필, 브라우저 자동화, 스킬 — 그리고 `localhost:3100`에서 열립니다
- **자연어 CRM**: "직원이 5명 이상인 회사를 보여줘"라고 물으면 라이브로 뷰가 업데이트됩니다. 수동 필터가 필요 없습니다
- **완전한 브라우저 자동화**: Chrome 프로필을 복사하여 에이전트가 동일한 인증 상태를 갖습니다 — "HubSpot에서 모든 것을 가져와"라고 하면 로그인하고, 내보내고, 가져옵니다
- **다중 뷰**: 테이블, 칸반, 캘린더, 타임라인(간트), 갤러리, 리스트 뷰 — 모두 에이전트가 YAML을 통해 구성 가능합니다
- **앱 빌더**: OpenClaw가 데이터에 접근할 수 있는 독립적인 웹 앱(대시보드, 도구, 게임)을 워크스페이스 내에서 빌드합니다
- **파일 시스템 우선**: 테이블 필터, 뷰, 열 토글, 캘린더 설정 — 모든 것이 파일이므로 OpenClaw가 직접 읽고 수정할 수 있습니다
- **코딩 에이전트로 작동**: DenchClaw가 DenchClaw를 빌드했습니다. Mac용 전체 파일 트리 브라우저이자 코드 에디터이기도 합니다

## 설정 방법

1. 원커맨드로 설치:

```bash
npx denchclaw
```

2. 온보딩 마법사를 완료합니다. DenchClaw는 `dench`라는 전용 OpenClaw 프로필을 생성하고 포트 19001에서 게이트웨이를 시작합니다.

3. 브라우저에서 `localhost:3100`을 엽니다. Safari에서는 PWA로 Dock에 추가하세요.

4. 대화를 시작하세요:

```text
"Leads" 객체를 생성해줘. 필드는: Name, Email, Company, Status (New/Contacted/Qualified/Won/Lost), Notes. Apollo에서 다운로드한 리드 CSV를 가져와줘.
```

```text
Status가 "Contacted"인 모든 리드를 보여주고 마지막 업데이트 순으로 정렬해줘. Status별로 그룹화된 칸반 뷰로 전환해줘.
```

```text
내 LinkedIn으로 가서 최근 내 프로필을 본 20명을 찾아서, 회사 정보를 보강해서 리드로 추가해줘.
```

```text
"New" 상태의 각 리드에게 최근 뉴스를 기반으로 개인화된 아웃리치 이메일 초안을 작성해줘. 새 "Outreach Drafts" 문서에 초안을 저장해줘.
```

5. Telegram이나 다른 연결된 메시징 플랫폼에서도 사용할 수 있습니다 — 이동 중에 휴대폰으로 파이프라인을 관리하세요.

## 필요한 스킬

DenchClaw는 필요한 모든 스킬을 자동으로 번들합니다:

- **CRM 스킬** — 객체, 필드, 엔트리, 다중 뷰 타입을 갖춘 DuckDB 기반 구조화된 데이터 관리
- **앱 빌더 스킬** — 데이터베이스에 접근할 수 있는 웹 앱(대시보드, 도구)을 워크스페이스 내에 빌드
- **브라우저 자동화 스킬** — 웹 스크래핑, 가져오기, 아웃리치를 위한 기존 Chrome 인증 상태를 갖춘 Chromium 브라우저

추가 스킬 설치나 구성이 필요 없습니다.

## 핵심 인사이트

- **파일 시스템 = 에이전트 네이티브 UI**: 모든 설정, 필터, 뷰가 YAML/마크다운 파일로 저장되기 때문에, OpenClaw는 코드를 편집하는 것처럼 자연스럽게 UI를 수정할 수 있습니다. API 래퍼가 필요 없습니다.
- **DuckDB가 최적의 선택**: 완전한 SQL을 지원하면서도 가장 작고 성능이 좋은 임베디드 데이터베이스입니다. 서버 프로세스, 자격증명, 네트워크 없음 — 그냥 파일입니다.
- **Chrome 프로필 복제는 슈퍼파워**: OAuth 흐름과 API 속도 제한과 싸우는 대신, DenchClaw는 브라우저의 인증 상태를 복사합니다. 에이전트가 당신이 보는 것을 보고, 당신이 하는 것을 합니다.
- **하나의 `npx` 명령이 주말 설정을 이김**: 전체 스택(데이터베이스, 웹 UI, OpenClaw 프로필, 게이트웨이, 브라우저, 스킬)이 스스로 설치하고 구성됩니다. Docker 없음, env 파일 없음, 의존성 지옥 없음.
