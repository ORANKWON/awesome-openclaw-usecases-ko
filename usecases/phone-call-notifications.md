# 전화 알림

에이전트는 이미 주식, 이메일, 스마트 홈, 캘린더 등을 모니터링하고 있지만, 알림은 무시하기 쉽습니다. 푸시 알림은 쌓이고, 채팅 메시지는 묻힙니다. 정말 중요한 것에는 스와이프로 지울 수 없는 무언가가 필요합니다.

이 활용 사례는 에이전트에게 전화를 알림 채널로 제공합니다. 충분히 긴급하면 에이전트가 실제 전화번호로 전화를 걸어 상황을 알려주고, 당신은 대화할 수 있습니다. 양방향 대화지, 자동 음성 전화가 아닙니다.

## 기능 설명

- 에이전트가 주목할 만한 일을 판단 (가격 알림, 긴급 이메일, 약속 리마인더 등)
- 에이전트가 [clawr.ing](https://clawr.ing)를 통해 전화를 겁니다 — Twilio 설정 불필요, API 키 구성 불필요
- 전화를 받으면 알림을 듣고 실시간으로 후속 질문을 할 수 있습니다
- 대화가 끝나면 통화가 종료됩니다

핵심 아이디어: 에이전트가 당신에게 전화합니다. 당신이 에이전트에게 전화하는 것이 아닙니다. 이것은 하트비트 체크, 크론 작업, 또는 모든 트리거와 함께 작동합니다 — 에이전트가 전화할 가치가 있는지 평가하고 실행합니다.

## 작동 원리

OpenClaw 에이전트는 이미 주도권을 가지고 있습니다 — 하트비트, 크론, 이벤트 트리거. 하지만 출력 채널은 확인하지 않을 수 있는 채팅 플랫폼으로 제한됩니다. 전화는 특히 책상에서 떨어져 있을 때 확실히 주목을 끄는 유일한 알림 채널입니다.

clawr.ing이 전화 인프라를 처리합니다. 설정 프롬프트 하나만 붙여넣으면 에이전트가 전화할 수 있는 능력을 얻습니다. Twilio 계정, 전화번호 프로비저닝, 웹훅 구성 불필요. 이 서비스는 100개 이상의 국가에서 실제 PSTN 전화를 지원합니다 (VoIP 오버레이가 아님).

## 필요한 스킬

- [clawr.ing](https://clawhub.ai/marcospgp/clawring) — [clawr.ing 대시보드](https://clawr.ing)에서 설정 프롬프트를 OpenClaw 채팅에 붙여넣으면 설치됩니다. CLI 설치 불필요.

이게 전부입니다. 다른 의존성 없습니다. 설정 프롬프트에는 API 키와 스킬 문서 링크가 포함되어 있으며 — 에이전트가 읽고 나머지를 알아냅니다.

## 예시: 모닝 브리핑 전화

매일 아침 맞춤형 브리핑으로 전화하는 크론 작업을 설정하세요:

```
평일 오전 7시 30분마다 모닝 브리핑으로 전화해. 포함할 내용:
- 우리 도시 날씨 예보
- 오늘 내 캘린더
- 밤새 온 긴급 이메일
- 내 관심사와 관련된 주요 뉴스 헤드라인 3개

간결하게 — 2분 이내로. 질문하면 답해줘.
안 받으면 재시도하지 마.
```

## 예시: 가격 알림

에이전트에게 무엇을 감시하고 언제 전화할지 알려주세요:

```
NVDA 주가를 모니터링해. 하루에 5% 이상 떨어지면
즉시 전화해서 무슨 일이 있었는지 알려줘. 하락을 설명할 수 있는
관련 뉴스도 포함해줘.
```

## 예시: 긴급 이메일 필터

```
업무 시간 동안 15분마다 받은편지함을 확인해.
상사나 긴급 표시된 이메일을 보면
요약해서 전화해줘. 나머지는 채팅 메시지만 보내.
```

## 핵심 인사이트

- **과도하게 사용하지 마세요.** 핵심은 전화가 "정말 중요하다"는 의미라는 것입니다. 에이전트가 하루에 10번 전화하면 무시하기 시작할 것입니다. 전화할 만한 것과 채팅할 만한 것의 명확한 임계값을 설정하세요.
- **하트비트나 크론과 함께 사용하세요.** clawr.ing은 전달 채널입니다 — 여전히 트리거가 필요합니다. 하트비트 체크(30분마다)는 모니터링 작업에 적합합니다. 크론 작업은 예약된 브리핑에 적합합니다.
- **양방향 대화.** 푸시 알림과 달리 전화에서 후속 질문을 할 수 있습니다. "잠깐, 어떤 이메일?" 또는 "지금 현재 가격은?" — 에이전트가 실시간으로 응답합니다.
- **전화용 빠른 모델.** 전화 대화는 빠른 응답이 필요합니다. OpenClaw가 스킬 수준 모델 라우팅을 지원하면 clawr.ing에 빠른 모델(Haiku 클래스)을 할당하세요. 생각하는 소리가 지연을 커버하지만, 빠를수록 항상 좋습니다.
- **프라이버시.** clawr.ing은 녹음이나 녹취록을 저장하지 않습니다. 오디오는 전송 중 암호화되고 처리 후 폐기됩니다.
