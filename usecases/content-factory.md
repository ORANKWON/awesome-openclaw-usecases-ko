# 콘텐츠 팩토리

## 문제점
콘텐츠 창작에는 리서치, 작성, 디자인의 세 단계가 있는데, AI 도구마다 하나씩 수동으로 프롬프트를 입력해야 합니다.

## 기능 설명
Discord에서 동작하는 멀티에이전트 콘텐츠 팩토리:
- **리서치 에이전트** (`#research`): 매일 오전 8시 트렌드 기사 및 경쟁사 콘텐츠 스캔
- **작성 에이전트** (`#scripts`): 최고 아이디어를 받아 전체 스크립트/스레드/뉴스레터 작성
- **썸네일 에이전트** (`#thumbnails`): AI 썸네일/커버 이미지 생성
- 스케줄에 따라 자동 실행 — 아침에 일어나면 완성된 콘텐츠가 준비되어 있음

## 설정 프롬프트

```
Discord 안에 콘텐츠 팩토리를 만들어 주세요.
에이전트별 채널을 설정하세요:

1. 리서치 에이전트 (#research): 매일 오전 8시에 내 니치의
   트렌드 기사, 경쟁사 콘텐츠, 소셜 미디어에서 잘 되고 있는 것을
   리서치하여 출처와 함께 상위 5개 콘텐츠 기회를 게시하세요.

2. 작성 에이전트 (#scripts): 리서치 에이전트의 최고 아이디어를 받아
   전체 스크립트/스레드/뉴스레터 초안을 작성하여 #scripts에 게시하세요.

3. 썸네일 에이전트 (#thumbnails): AI 썸네일 또는 커버 이미지를 생성하여
   #thumbnails에 게시하세요.

매일 아침 이 파이프라인을 자동으로 실행하세요.
```

## 필요한 스킬
- Discord 멀티채널
- `sessions_spawn` / `sessions_send`
- `x-research-v2`
- 로컬 이미지 생성
- knowledge-base 스킬
