# n8n 워크플로우 오케스트레이션

## 문제점
OpenClaw가 모든 것을 직접 처리할 때: 구축한 내용이 보이지 않고, API 키마다 자격증명이 분산되며, 결정론적 서브태스크에 LLM 토큰이 낭비됩니다.

## 기능 설명
프록시 패턴 — OpenClaw가 인바운드 웹훅이 있는 n8n 워크플로우를 생성한 후, 모든 외부 API 상호작용에 해당 웹훅을 호출합니다. 에이전트는 자격증명을 직접 다루지 않습니다.

```
OpenClaw → 웹훅 호출 (자격증명 없음) → n8n 워크플로우 (잠금, API 키 보유) → 외부 서비스
```

**장점:**
- 자격증명이 n8n 자격증명 저장소에 보관
- n8n 드래그앤드롭 UI로 시각적 디버깅
- 테스트 후 워크플로우 잠금 가능
- n8n에서 유효성 검사, 속도 제한, 승인 게이트 추가 가능

## AGENTS.md 패턴

```
외부 API와 상호작용할 때:
1. 환경이나 스킬 파일에 API 키 절대 저장하지 마세요
2. n8n 워크플로우가 이미 존재하는지 확인
3. 없으면 n8n API로 웹훅 트리거가 있는 워크플로우 생성
4. 사용자에게 자격증명 추가 및 워크플로우 잠금 요청
5. 이후 모든 호출은 JSON 페이로드와 함께 웹훅 URL 사용

워크플로우 이름 규칙: openclaw-{서비스}-{액션}
```
