# arXiv 논문 리더

arXiv 논문을 읽는다는 것은 PDF를 다운로드하고, 논문 간 전환 시 맥락을 잃고, 조밀한 LaTeX 표기법을 파싱하는 데 어려움을 겪는 것을 의미합니다. 워크스페이스를 떠나지 않고 대화형으로 논문을 읽고, 분석하고, 비교하고 싶습니다.

이 워크플로우는 에이전트를 연구 독서 어시스턴트로 만듭니다:

- arXiv ID로 논문을 가져오고 깔끔하고 읽기 쉬운 텍스트를 받습니다 (LaTeX가 자동으로 평탄화됨)
- 먼저 논문 구조를 탐색 — 전체 텍스트를 읽기 전에 섹션을 나열하여 무엇을 읽을지 결정합니다
- 여러 논문의 초록을 빠르게 스캔하여 읽기 목록을 분류합니다
- 에이전트에게 특정 섹션을 요약, 비교 또는 비평하도록 요청합니다
- 결과가 로컬에 캐시됩니다 — 논문을 다시 방문하는 것이 즉각적입니다

## 필요한 스킬

- [arxiv-reader](https://github.com/Prismer-AI/Prismer/tree/main/skills/arxiv-reader) 스킬 (3개 도구: `arxiv_fetch`, `arxiv_sections`, `arxiv_abstract`)

Docker나 Python이 필요 없습니다 — 스킬은 Node.js 내장 기능을 사용하여 독립적으로 실행됩니다. arXiv에서 직접 다운로드하고, LaTeX 소스의 압축을 풀고, include를 자동으로 평탄화합니다.

## 설정 방법

1. [Prismer 저장소](https://github.com/Prismer-AI/Prismer/tree/main/skills/arxiv-reader)에서 `arxiv-reader` 스킬을 설치합니다 — `skills/arxiv-reader/` 디렉토리를 OpenClaw 스킬 폴더에 복사하세요.

2. 스킬을 사용할 준비가 되었습니다. OpenClaw에 프롬프트하세요:
```text
[주제]를 연구하고 있어. 내 워크플로우는 이래:

1. arXiv ID를 주면 (예: 2301.00001):
   - 먼저 초록을 가져와서 관련성이 있는지 결정할 수 있게 해줘
   - "읽어줘"라고 하면 전체 논문을 가져와줘 (기본적으로 부록 제거)
   - 주요 기여, 방법론, 결과를 요약해줘

2. 여러 ID를 주면:
   - 모든 초록을 가져와서 비교 표를 만들어줘
   - 내 연구 주제와의 관련성으로 순위를 매겨줘

3. 특정 섹션에 대해 물어보면:
   - 먼저 논문의 섹션을 나열해줘
   - 그런 다음 관련 섹션을 가져와서 자세히 설명해줘

내가 읽은 논문과 핵심 요점의 목록을 계속 유지해줘.
```

3. 시도해보세요: "2401.04088을 읽어줘 — 주요 기여가 뭐야?"
