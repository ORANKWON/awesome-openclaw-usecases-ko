# 기여 가이드

## 제출 방법

1. `/usecases/` 폴더에 활용 사례 이름으로 마크다운 파일 생성
2. 아래 섹션을 포함:
   - 문제점 (Pain Point)
   - 기능 설명 (What It Does)
   - 프롬프트 (Prompts)
   - 필요한 스킬 (Skills Needed)
   - 관련 링크 (Related Links)
3. `README.md`의 해당 카테고리 표에 항목 추가
4. Pull Request 열기

## 규칙

- 파일 하나에 사례 하나
- 설명은 간결하되 재현 가능할 만큼 충분히 작성
- 새 카테고리가 필요하면 제안 가능
- 접근 방식이 의미 있게 다르다면 중복 사례도 허용
- **암호화폐 관련 사례는 받지 않습니다**
- **AI로 사례를 생성하지 마세요. 직접 테스트하고 검증한 사례만 제출하세요**
